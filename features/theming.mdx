Bodequita uses **Material Design 3** (Material You) with two hand-crafted colour schemes — light and dark — built on a green-centric brand palette. The active theme persists between app sessions using `SharedPreferences`.

## Switching the Theme

Users can toggle between light and dark mode from the **Profile** screen:

<Steps>
  <Step title="Open the Profile screen">
    Tap **Perfil** in the navigation drawer.
  </Step>
  <Step title="Locate the Theme setting">
    Scroll to the **Configuraciones** card. The **Tema** row shows the current state (e.g., *"Modo claro activado"* or *"Modo oscuro activado"*) and a toggle switch.
  </Step>
  <Step title="Toggle the theme">
    Tap the switch (or tap anywhere on the Tema row) to flip between light and dark mode. The entire app re-renders immediately with the new colour scheme.
  </Step>
</Steps>

The change takes effect instantly across all screens because the theme state is held in a `StateFlow` that the root `MiniTiendaTheme` composable observes.

## ThemeManager

`ThemeManager` is a utility class that owns the theme preference:

```kotlin
class ThemeManager(context: Context) {
    private val prefs = context.getSharedPreferences("mini_tienda_theme", Context.MODE_PRIVATE)
    private val THEME_KEY = "is_dark_theme"

    private val _isDarkTheme = MutableStateFlow(getSavedTheme())
    val isDarkTheme: StateFlow<Boolean> = _isDarkTheme.asStateFlow()

    fun toggleTheme() {
        val newTheme = !_isDarkTheme.value
        _isDarkTheme.value = newTheme
        prefs.edit().putString(THEME_KEY, newTheme.toString()).apply()
    }

    fun setTheme(isDark: Boolean) {
        _isDarkTheme.value = isDark
        prefs.edit().putBoolean(THEME_KEY, isDark).apply()
    }
}
```

- **Default**: Light mode (`false`) when the app is first installed.
- **Persistence**: The preference is read from `SharedPreferences` on startup so the last-used theme is restored automatically.
- **API**: `toggleTheme()` flips the current value; `setTheme(isDark)` sets it directly.

The `ThemeManager` instance is created in `MainActivity` and passed through `MainNavigation` to `ProfileScreen`.

## Material Design 3 Integration

The `MiniTiendaTheme` composable wraps `MaterialTheme` with the selected colour scheme:

```kotlin
@Composable
fun MiniTiendaTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    content: @Composable () -> Unit
) {
    val colorScheme = if (darkTheme) DarkColorScheme else LightColorScheme
    // Status bar colour synced to primary
    MaterialTheme(
        colorScheme = colorScheme,
        typography = Typography,
        content = content
    )
}
```

- The **status bar** colour is updated to match `colorScheme.primary` via a `SideEffect`.
- Status bar icon lightness is inverted for dark mode (`isAppearanceLightStatusBars = !darkTheme`).
- All Material 3 components (`Card`, `Button`, `TextField`, `Switch`, `NavigationDrawerItem`, etc.) automatically pick up the active colour scheme tokens.

<Info>
  The `darkTheme` parameter defaults to `isSystemInDarkTheme()` but the `ThemeManager` preference takes precedence when passed explicitly from `MainActivity`, so user choice overrides the system setting.
</Info>
